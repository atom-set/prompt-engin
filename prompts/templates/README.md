# 提示词模板

本目录包含通用的提示词模板，可以在不同场景下复用。

## 模板列表

| 模板文件 | 说明 | 适用场景 |
|---------|------|---------|
| `project-template.md` | 项目模板 | 创建新项目时使用 |
| `rule-modularization-guide.md` | 规则模块化指南 | 规则文件拆分和模块化时使用 |
| `stage-template.md` | 阶段模板 | 创建新阶段规则时使用 |
| `technical-solution-template.md` | 技术方案模板 | 生成技术方案文档时使用 |

## 模板详细说明

### project-template.md

项目模板，用于创建新项目时的提示词模板。

### rule-modularization-guide.md

规则模块化指南，提供规则文件拆分和模块化的指导，包括：
- 文件大小约束
- 功能单一性要求
- 可扩展性要求
- 目录组织要求
- 索引文件要求

### stage-template.md

阶段模板，用于创建新阶段规则时的提示词模板。

### technical-solution-template.md

技术方案最终版本提示词模板，用于指导生成结构化的技术方案文档。

**模板结构**：
- 12个主要章节（概述、需求背景、架构设计、设计方案、流程图设计、前端实现、Node服务端实现（可选）、关键技术实现细节、监控和日志设计、风险评估、代码改动总结、总结）
- 完整的格式规范说明
- 占位符说明和使用指南
- 最佳实践建议

**使用场景**：
- 生成技术方案文档
- 技术方案评审
- 技术方案归档

**特点**：
- ✅ 通用性：使用占位符，适用于不同项目
- ✅ 灵活性：可选章节可根据需要包含或跳过
- ✅ 完整性：覆盖技术方案文档的主要要素
- ✅ 规范性：提供格式规范和最佳实践

## 使用方式

这些模板可以作为基础，根据具体需求进行定制：

1. **选择合适的模板**：根据使用场景选择对应的模板文件
2. **替换占位符**：将模板中的占位符（如 `{项目名称}`）替换为实际内容
3. **调整章节**：根据项目实际情况，调整或跳过可选章节
4. **补充内容**：按照模板要求，补充详细的技术细节和实现说明

## 注意事项

- 所有模板都遵循项目的格式规范（文档格式、时间格式等）
- 使用模板时，必须替换所有占位符，不能保留占位符文本
- 时间字段必须使用实际时间，禁止使用假设日期
- 代码示例应完整可运行，避免片段代码
