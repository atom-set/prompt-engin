## 一、分享脉络

### 1.1 核心逻辑线

```
问题发现 → 动机分析 → 解决方案 → 技术实现 → 实际应用 → 未来展望
```

### 1.2 详细脉络

#### 第一步：建立问题意识（引起共鸣）
- **目标**：让听众意识到问题的存在和严重性
- **方式**：通过提问和痛点分析
- **预期效果**：听众产生"我也有这个问题"的共鸣

#### 第二步：说明动机和初衷（建立信任）
- **目标**：说明为什么做这个项目，建立项目可信度
- **方式**：展示核心动机和设计理念
- **预期效果**：听众理解项目的价值和意义

#### 第三步：展示核心价值（证明价值）
- **目标**：说明解决了什么问题，带来什么价值
- **方式**：功能展示 + 数据对比
- **预期效果**：听众认可项目的价值

#### 第四步：技术实现（建立信心）
- **目标**：简要说明如何实现，建立技术信心
- **方式**：架构介绍 + 使用流程
- **预期效果**：听众认为项目可行、可靠

#### 第五步：实际应用（增强说服力）
- **目标**：通过实际案例证明效果
- **方式**：使用场景 + 实际案例
- **预期效果**：听众看到实际效果，增强使用意愿

#### 第六步：未来展望（展示能力）
- **目标**：说明未来希望实现的能力
- **方式**：展示3个核心能力方向
- **预期效果**：听众了解项目未来发展方向

#### 第七步：Q&A 和总结（强化记忆）
- **目标**：解答疑问，强化核心价值
- **方式**：互动问答 + 价值总结
- **预期效果**：听众记住核心价值，产生行动意愿

## 二、结构框架

### 2.1 金字塔模型原则

**核心原则**：
1. **结论先行**：每个章节先给出结论
2. **以上统下**：上层结论是下层论据的总结
3. **归类分组**：同类论据归为一组，**每组不超过3个**
4. **逻辑递进**：按照重要性或时间顺序排列

**应用要求**：
- ✅ 每个论据组不超过3个要点
- ✅ 同类内容归类合并
- ✅ 逻辑清晰，层次分明

### 2.2 整体结构（8 个章节）

```
一、开场：问题与痛点
  ├─ 1.1 AI 辅助开发的现状
  └─ 1.2 技术团队的痛点（3 个核心痛点）✅

二、工程初衷：为什么做这个项目
  ├─ 2.1 核心动机（3 个动机）✅
  ├─ 2.2 设计理念：为什么这样设计
  └─ 2.3 项目定位

三、核心价值：解决了什么问题
  ├─ 3.1 核心功能（3 个功能）✅
  └─ 3.2 实际效果（数据对比）

四、技术实现：如何做到的
  ├─ 4.1 技术架构
  └─ 4.2 使用流程

五、使用场景：实际应用
  ├─ 5.1 典型使用场景（3 个场景）✅
  └─ 5.2 实际案例（3 个案例）✅

六、未来展望：希望实现的能力
  ├─ 6.1 基于用户角色的个性化生成 ⭐
  ├─ 6.2 AI 辅助规则生成
  └─ 6.3 规则效果分析和优化

七、Q&A
  └─ 7.1 常见问题

八、总结
  ├─ 8.1 核心价值回顾
  └─ 8.2 行动建议
```

**说明**：✅ 标记表示已按金字塔模型调整（每组不超过3个）

### 2.3 章节层次

#### 第一层：主要章节（8 个）
- 每个章节对应一个核心主题
- 章节之间逻辑递进

#### 第二层：小节（2-3 个小节）
- 每个小节聚焦一个子主题
- 小节之间相互支撑

#### 第三层：要点（**不超过3个要点**）✅
- 每个要点说明一个具体内容
- 要点之间并列或递进
- **符合金字塔模型：每组不超过3个**

## 三、章节关系

### 3.1 章节逻辑关系

```
一、问题与痛点
    ↓ （问题驱动）
二、工程初衷
    ↓ （动机驱动）
三、核心价值
    ↓ （价值证明）
四、技术实现
    ↓ （技术支撑）
五、使用场景
    ↓ （应用证明）
六、未来展望
    ↓ （愿景展示）
七、Q&A
    ↓ （互动强化）
八、总结
```

### 3.2 关键连接点

#### 连接点1：问题 → 动机
- **问题**：提示词分散和同步困难（最核心痛点）
- **动机**：解决提示词分散和同步的痛点（核心动机）
- **关系**：问题驱动动机，动机对应问题

#### 连接点2：动机 → 价值
- **动机**：解决提示词分散和同步的痛点
- **价值**：集中管理和结构化组织（解决核心痛点）
- **关系**：动机转化为价值，价值实现动机

#### 连接点3：价值 → 技术
- **价值**：集中管理、多平台支持、Token 优化和个性化适配
- **技术**：模块化设计、多格式支持、Token 优化方案、技能系统
- **关系**：技术支撑价值，价值指导技术

#### 连接点4：技术 → 应用
- **技术**：提供工具和方案
- **应用**：实际使用场景和案例
- **关系**：技术实现应用，应用验证技术
