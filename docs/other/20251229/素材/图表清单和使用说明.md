## 🔧 图表转换说明

### 转换工具

根据项目文档，可以使用以下工具转换 Mermaid 图表：

1. **单个图表转换**：`scripts/utils/mermaid_to_png.sh`
2. **批量转换**：`scripts/utils/extract_and_convert_mermaid.py`

**注意**：如果工具不存在，需要先创建或使用在线工具转换。

### 转换步骤

#### 方法1：使用批量转换工具（推荐）

```bash
# 批量提取并转换所有图表
python3 scripts/utils/extract_and_convert_mermaid.py \
    docs/other/20251229/完整分享内容.md \
    docs/other/20251229/images/
```

**优点**：
- 自动提取所有图表
- 根据章节标题自动命名
- 批量转换，效率高

#### 方法2：手动转换单个图表

1. **提取图表代码**：从文档中复制 Mermaid 代码块
2. **保存为 .mmd 文件**：保存到临时目录
3. **使用转换工具**：
   ```bash
   bash scripts/utils/mermaid_to_png.sh chart.mmd docs/other/20251229/images/ chart-name
   ```

#### 方法3：使用在线工具

如果本地工具不可用，可以使用在线 Mermaid 编辑器：
- Mermaid Live Editor：https://mermaid.live/
- 导出为 PNG 格式（确保使用白色背景）

## 📝 图表使用建议

### 在演示中使用

1. **开场展示**：使用"分享逻辑脉络图"展示整体结构
2. **讲解设计理念**：使用"设计理念流程图"帮助理解
3. **数据对比**：使用"Token 优化效果对比图"和"团队协作场景对比图"增强说服力
4. **技术讲解**：使用"技术架构图"和"使用流程图"帮助理解

### 在文档中使用

1. **保持图表位置**：图表应紧跟在相关说明文字后
2. **添加图表说明**：每个图表前应有清晰的标题和说明
3. **使用相对路径**：如果转换为图片，使用相对路径引用

### 图表优化建议

1. **颜色方案**：
   - 使用统一的颜色方案（已在图表中定义）
   - 核心模块使用强调色（浅黄色）
   - 重要节点使用绿色（表示优化/成功）
   - 问题节点使用红色（表示需要改进）

2. **图表大小**：
   - 确保图表在演示时清晰可见
   - 避免节点过多导致图表过于复杂
   - 必要时拆分为多个小图表

3. **文字说明**：
   - 图表标题清晰表达图表用途
   - 关键节点添加简要说明
   - 复杂图表后添加文字解释
