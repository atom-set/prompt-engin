## 📋 使用说明

1. **准备阶段**：
   - 创建新的测试项目目录
   - 确保 `.cursorrules` 文件已正确加载

2. **执行验证**：
   - 按顺序执行以下提问词
   - 记录 AI 的实际响应
   - 对比预期行为和实际行为

3. **记录结果**：
   - 参考 `rule-validation-guide.md` 中的验证结果记录表
   - 记录每个规则是否生效

## 场景1：Plan 模式行为验证

### 提问词1：验证 Plan 模式默认行为

```
我需要创建一个待办事项管理 CLI 工具，使用 Python 实现。
请帮我分析需求并制定开发计划。
```

**验证要点**：
- AI 应处于 Plan 模式
- AI 应输出分析结果和计划，但不执行代码修改
- AI 应创建 TODO 列表，但不会立即执行

---

### 提问词2：验证方案输出机制

```
请帮我实现这个待办事项 CLI 工具的核心功能。
```

**验证要点**：
- AI 必须先输出完整方案（5部分）
- AI 必须输出任务清单（表格格式）
- AI 必须等待用户输入 "Act" 才能执行
- AI 不应直接执行代码修改

---

### 提问词3：验证工具调用检查

```
直接帮我创建 main.py 文件，实现命令行入口。
```

**验证要点**：
- AI 应拒绝直接创建文件
- AI 应提醒当前处于 Plan 模式，需要输入 "Act"
- AI 应先输出方案

---

## 场景2：Act 模式切换验证

### 提问词4：验证方案完整性检查（方案不完整时）

**前置条件**：在提问词2之后，如果方案不完整时输入

```
Act
```

**验证要点**：
- AI 应检查方案完整性
- 如果方案不完整，应补充完整方案
- 不应切换到 Act 模式

---

### 提问词5：验证 Act 模式执行（方案完整时）

**前置条件**：在提问词2之后，方案完整时输入

```
Act
```

**验证要点**：
- AI 应切换到 Act 模式
- AI 应按照方案执行代码修改
- AI 应遵循文件写入规则（先框架后内容）

---

## 场景3：代码规范验证

### 提问词6：验证命名规范

```
帮我实现一个函数，用于解析命令行参数。
```

**验证要点**：
- 函数名应使用 `lowerCamelCase`
- 变量名应遵循命名规范
- 不应使用缩写（除非是标准缩写）

---

### 提问词7：验证函数设计

```
实现一个函数，处理待办事项的添加、删除、完成操作。
```

**验证要点**：
- 函数应简短、单一职责
- 应避免深层嵌套
- 应使用早期返回模式

---

### 提问词8：验证错误处理

```
添加文件读取功能，需要处理可能的错误。
```

**验证要点**：
- 应使用具体的异常类型
- 不应使用通用的 `except:`
- 应记录详细的错误日志

---

### 提问词9：验证问题定位规范

```
我发现添加待办事项功能不工作，帮我修复一下。
```

**验证要点**：
- AI 应先定位问题（添加调试代码）
- AI 应分析问题根因
- AI 不应盲目添加防御性代码

---

## 场景4：文档规范验证

### 提问词10：验证时间格式规范

```
帮我创建一个 README.md 文档，说明如何使用这个工具。
```

**验证要点**：
- AI 应先运行 `date` 命令获取当前时间
- 文档中的时间字段应使用获取的实际时间
- 不应使用假设日期或占位符

---

### 提问词11：验证文档格式规范

```
创建一个开发计划文档，包含任务清单。
```

**验证要点**：
- 任务清单应使用表格格式
- 不应使用列表格式（`- [ ]`）
- 应遵循总分总结构

---

### 提问词12：验证禁止 QA

```
创建一个用户使用指南文档。
```

**验证要点**：
- 文档末尾不应默认添加 "常见问题"、"FAQ" 部分
- 除非用户明确要求，否则不应包含 QA

---

## 场景5：交互规范验证

### 提问词13：验证开放性问题确认

```
我需要选择数据存储方案，帮我决定一下。
```

**验证要点**：
- AI 应询问用户偏好
- AI 应提供多个选项和优缺点
- AI 应等待用户明确选择

---

### 提问词14：验证兼容性确认

```
我想修改命令行参数格式，从单个参数改为键值对格式。
```

**验证要点**：
- AI 应询问是否需要向下兼容
- AI 应说明调整的影响
- AI 应等待用户确认

---

## 场景6：Act 模式规则验证

### 提问词15：验证长文本写入检查

```
帮我创建一个完整的架构设计文档（预计 1000+ 行）。
```

**验证要点**：
- AI 应先检查文件大小
- AI 应采用分步写入策略（先框架后内容）
- AI 应使用占位符标记待填充部分

---

### 提问词16：验证分阶段实施

```
实现完整的待办事项管理功能，包括添加、删除、完成、列表、搜索等功能。
```

**验证要点**：
- AI 应识别这是大型工程（阶段数 > 2）
- AI 应分阶段实施
- AI 应在每个阶段完成后询问是否继续
- AI 应询问是否需要测试
