# 提示词规则验证快速开始指南

> **文档说明**：快速开始验证提示词规则的效果
> **创建时间**：2025-12-15（本地时间）

---

## 🚀 5 分钟快速开始

### 第一步：创建测试项目

```bash
# 创建测试项目目录
mkdir todo-cli-validation
cd todo-cli-validation

# 复制规则文件（从 prompt-engin 项目）
cp /path/to/prompt-engin/.cursorrules .
```

### 第二步：打开项目

在 Cursor 中打开 `todo-cli-validation` 目录，确保 `.cursorrules` 文件已正确加载。

### 第三步：执行第一个验证

复制以下提问词，向 AI 提问：

```
我需要创建一个待办事项管理 CLI 工具，使用 Python 实现。
请帮我分析需求并制定开发计划。
```

### 第四步：验证结果

检查 AI 的响应：

- ✅ **模式标识**：响应开头应显示 `# 模式：Plan`
- ✅ **只输出方案**：AI 应输出分析结果和计划，但不执行代码修改
- ✅ **创建 TODO**：AI 应创建 TODO 列表，但不会立即执行

如果符合预期，说明 Plan 模式规则生效 ✅

## ❓ 常见问题

### Q1：如何确认规则文件已加载？

**A**：在 Cursor 中，AI 的响应开头应显示模式标识（如 `# 模式：Plan`），说明规则已加载。

### Q2：验证时需要注意什么？

**A**：
- 每个提问词应独立执行
- 记录 AI 的完整响应
- 对比预期行为和实际行为

### Q3：如果规则未生效怎么办？

**A**：
- 检查 `.cursorrules` 文件是否正确加载
- 检查规则文件内容是否完整
- 记录具体场景和原因，便于后续改进
